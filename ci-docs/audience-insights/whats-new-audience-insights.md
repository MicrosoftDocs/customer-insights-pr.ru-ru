## <a name="february-2022-updates"></a>Обновления за февраль 2022 г.

Обновления в феврале 2022 года включают новые функции, улучшения производительности и исправления ошибок.

### <a name="general-availability-for-prediction-models"></a>Общая доступность прогнозных моделей

Готовые прогнозные модели, включая **отток подписок**, **отток транзакций** и **ценность клиента на протяжении жизненного цикла** становятся общедоступными в составе Customer Insights. 

Дополнительные сведения см. в разделе [Обзор прогнозов](predictions-overview.md).

### <a name="new-data-source-integration-with-azure-synapse-analytics-preview"></a>Новый источник данных: интеграция с Azure Synapse Analytics (предварительная версия)

Azure Synapse Analytics — это служба корпоративной аналитики, которая ускоряет получение аналитик из хранилищ данных и систем больших данных.

Если ваша организация уже использует расширенные аналитические возможности Azure Synapse Analytics и храните выходные данные в базах данных Data Lake, вы можете легко передать эти данные в Customer Insights. Дополнительная информация дана в теме [Подключение источника данных Azure Synapse (предварительная версия)](connect-synapse.md).

### <a name="liveramp-enrichment-preview"></a>Обогащение LiveRamp (предварительная версия)

LiveRamp обеспечивает детерминированное автономное разрешение идентификации и консолидацию данных о клиентах. Вы можете сопоставить личные идентификаторы в ваших клиентских данных с графом идентификации AbiliTec и получать идентификаторы AbiliTec. Затем вы можете использовать эти идентификаторы для лучшей унификации данных ваших клиентов.

Для получения дополнительной информации см. [Обогащение профилей клиентов идентификационными данными из LiveRamp (предварительная версия)](enrichment-liveramp.md).

### <a name="enrichment-for-data-sources-preview"></a>Обогащение источников данных (предварительная версия)

Используйте данные из таких источников, как Microsoft и другие партнеры, чтобы обогатить данные о клиентах перед унификацией данных. Обогащения источников данных помогают повысить полноту и качество данных, что поможет добиться лучших результатов после унификации данных.

Дополнительная информация дана в теме [Обогащение источников данных (предварительная версия)](data-sources-enrichment.md).

### <a name="change-owner-of-environment"></a>Изменить владельца среды

Хотя права администратора в Customer Insights могут быть у нескольких пользователей, только один пользователь является владельцем среды. Улучшенный интерфейс позволяет менять владельцев среды и заявлять право собственности, если прежний владелец покинул организацию. 

Для получения дополнительной информации см. [Изменить владельца среды](manage-environments.md#change-the-owner-of-an-environment).

### <a name="data-preparation-process-lists-corruption-reason-for-corrupted-records"></a>В процессе подготовки данных указывается причина повреждения поврежденных записей

Процесс подготовки данных теперь показывает причину повреждения для всех полей с поврежденными данными на уровне отдельной записи для облегчения идентификации. 

Дополнительная информация дана в теме [Поврежденные источники данных](entities.md#corrupted-data-sources).

### <a name="end-of-preview-for-reporting-features-in-the-engagement-insights-capability"></a>Окончание предварительной версии функций отчетности в возможности аналитики взаимодействия

Предварительная версия возможности аналитики взаимодействий Dynamics 365 Customer Insights закончилась 15 февраля 2022 г.  
Это изменение означает, что пробная версия Customer Insights больше не включает возможность создавать воронки или другие функции отчетности.

Мы приглашаем вас изучить и оценить многие другие функции [Customer Insights](https://dynamics.microsoft.com/ai/customer-insights/), платформы данных клиентов Майкрософт (CDP).    
 
В течение переходного периода существующие участники предварительной версии по-прежнему имеют доступ к некоторым возможностям и функциям предварительной версии:

- Получите код для настройки веб-сайта или мобильного приложения 
- См. раздел события и свойства событий 
- Усовершенствуйте единые профили с помощью принятых и уточненных событий, чтобы извлечь выгоду из полной ценности своих клиентских данных.
  
В течение переходного периода захваченные события по-прежнему передаются в подключенное озеро данных Data Lake. После отключения этой функции обмен данными между аналитикой взаимодействия и аналитикой аудитории прекратится, и в подключенное хранилище не будут отправляться новые события.
Если у вас есть вопросы по поводу завершения предварительной версии возможностей, обратитесь непосредственно к вашей группе по работе с учетными записями Майкрософт. Ваша группа по работе с клиентами будет держать вас в курсе предстоящих запусков.

### <a name="advanced-data-unification-features"></a>Расширенные функции объединения данных

#### <a name="enable-conflict-resolution-policies-at-the-individual-attribute-level"></a>Включение политик разрешения конфликтов на уровне отдельных атрибутов

При дедупликации записей клиентов внутри сущности вам может не понадобиться выбирать полную запись в качестве победителя. Теперь мы позволяем вам объединять лучшие поля из разных записей на основе правил для каждого атрибута. Например, вы можете сохранить самую последнюю электронную почту И наиболее полный адрес из разных записей. 

Теперь вы можете определить отдельные правила слияния для отдельных атрибутов при дедупликации и слиянии записей в одной сущности. Раньше мы позволяли вам выбрать только одно правило слияния (сохранение записей на основе полноты данных о давности), и это правило применялось на уровне записи ко всем атрибутам. Это неидеально, когда некоторые данные, которые вы хотите сохранить, находятся в записи A, а другие полезные данные — в записи B.

Для получения дополнительной информации см. [Определите дедупликацию для соответствующей сущности](match-entities.md#define-deduplication-on-a-match-entity).

#### <a name="custom-rules-for-matching"></a>Пользовательские правила сопоставления

Бывают случаи, когда вам нужно указать исключение из общих правил, чтобы НЕ сопоставлять записи. Это может произойти, когда несколько человек обмениваются информацией, достаточной для того, чтобы система сопоставила их как одного человека. Например, близнецы с одинаковым фамилия, живущие в одном городе и с одинаковой датой рождения.

Исключения гарантируют, что неправильная унификация данных может быть устранена в правилах унификации. Вы можете добавить несколько исключений к правилу.

Дополнительные сведения см. в разделе [Добавить исключения в правило](match-entities.md#add-exceptions-to-a-rule).

#### <a name="provide-additional-conflict-resolution-policies-and-enable-grouping-of-attributes"></a>Предоставьте дополнительные политики разрешения конфликтов и включите группировку атрибутов.

Эта функция позволяет рассматривать группу полей как единое целое. Например, если наши записи содержат поля Address1, Address2, City, State и Zip. Скорее всего, мы не захотим объединять в Address2 другой записи, думая, что это сделает наши данные более полными.

Теперь вы можете объединить группу связанных полей и применить к группе единую политику слияния. 

Для получения дополнительной информации см. [Объединение группы полей](merge-entities.md#combine-a-group-of-fields).

### <a name="b-to-b"></a>Бизнес-бизнес

Начиная с октября 2021 года вы можете работать с корпоративными учетными записями и связанными с ними контактами в Customer Insights. Раньше приложение в основном ориентировалось на индивидуальных потребителей. Несколько функциональных областей были обновлены для поддержки сценариев "бизнес-бизнес" поверх нового типа среды. Обзор поддерживаемых функций "бизнес-бизнес" см. в разделе [Работа с организациями в аналитике аудитории](work-with-business-accounts.md).

В следующих разделах освещаются некоторые ключевые области, которые были адаптированы для поддержки корпоративных учетных записей и индивидуальных потребителей.

#### <a name="export-segments-based-on-business-accounts"></a>Экспорт сегментов на основе корпоративных учетных записей

Все экспорты сегментов в аналитике аудитории доступны в контексте корпоративных учетных записей. Для большинства видов экспорта сегментов требуется дополнительная конфигурация и [контактная информация, проецируемая](segment-builder.md#create-a-new-segment) в базовых сегментах, чтобы быть действительными для организаций. Дополнительные сведения см. в разделе [Экспорт сегментов](export-destinations.md#export-segments).

#### <a name="use-the-linkedin-ads-export-with-business-accounts"></a>Использование экспорта LinkedIn Ads с корпоративными учетными записями

Экспорт LinkedIn Ads теперь доступен для таргетинга на контакты и компании в контексте корпоративных учетных записей. При выборе таргетинга на компанию в качестве основного фокуса экспорта LinkedIn вы можете экспортировать сегменты, созданные на основе корпоративных учетных записей, без необходимости проецировать контактную информацию. Для получения дополнительной информации перейдите к документации об [экспорте LinkedIn Ads](export-linkedin-ads.md) и разнице между [таргетингом на контакты](https://business.linkedin.com/marketing-solutions/ad-targeting/contact-targeting) и [таргетингом на компанию](https://business.linkedin.com/marketing-solutions/ad-targeting/account-targeting). 

#### <a name="create-measures-based-on-business-accounts-and-their-hierarchy"></a>Создание мер на основе корпоративных учетных записей и их иерархии

Конструктор мер позволяет создавать меры для корпоративных учетных записей и при желании использовать информацию об иерархии. Информация об иерархии используется для объединения расчета мер по учетной записи и всем связанным с ней дочерними организациями. Например, вы можете создать такие меры, как общий доход, для каждой группы корпоративных учетных записей, определяемых их иерархией. Дополнительные сведения см. в разделе [Определение мер и управление ими](measures.md).

#### <a name="create-segments-based-on-business-accounts-and-their-hierarchy"></a>Создание сегментов на основе корпоративных учетных записей и их иерархии

Конструктор сегментов позволяет создавать сегменты корпоративных учетных записей, которые дополнительно могут включать контактную информацию для каждой учетной записи в сегменте. Если у вас настроена иерархия учетных записей, вы можете использовать информацию об иерархии учетных записей при создании сегментов. Дополнительные сведения см. в разделе [Создание нового сегмента](segment-builder.md#create-a-new-segment).

#### <a name="retain-your-business-accounts-with-deep-insights-to-their-churn-tendency"></a>Сохранение корпоративных учетных записей с глубокой аналитикой тенденции их оттока

Модель прогноза оттока клиентов теперь поддерживает и корпоративные учетные записи. Вы можете оценить риск оттока не только для учетной записи, но и для комбинации учетной записи и категории продукта или услуги, которые они покупают у вас. Это дополнение поможет вам понять, есть ли повышенная вероятность того, что организация перестанет покупать у вас в целом или только для определенной категории товаров или услуг. Чтобы еще больше помочь вам использовать эту модель искусственного интеллекта, в ней также перечислены причины, по которым возможен отток организации. Для получения дополнительной информации см. раздел [Прогноз оттока транзакций (предварительная версия)](predict-transactional-churn.md).

#### <a name="see-contacts-of-a-business-account-in-customer-view"></a>Просмотр контактов корпоративных учетных записей в представлении клиента

Если корпоративные учетные записи сопоставлены со связанными учетными записями, приложение Customer Insights показывает эти связанные контакты как часть представления сведений о клиенте. Дополнительную информацию см. в разделе [Профили клиентов](customer-profiles.md).

  Понимание того, что клиенты заинтересованы в покупке, — это одно из главных требований увеличения доходов бизнеса и повышения лояльности клиентов за счет персонализации и взаимодействия. Предоставление рекомендаций по продуктам, которые не соответствуют интересам вашего клиента, может создать ощущение разрыва между клиентом и вашей компанией и, в конечном итоге, ограничить общий потенциальный доход и опыт для клиента. 

  Используя свои собственные данные, теперь вы можете прогнозировать, какие товары ваши клиенты, скорее всего, купят в будущем. Для получения дополнительной информации см. [Прогноз рекомендации продуктов](predict-product-recommendation.md).

#### <a name="system-administration"></a>Системное администрирование

- **Копирование среды поддерживается для разных типов источников данных**

  Администраторы могут копировать конфигурации среды в новую среду в той же организации. Эта функция расширяет функциональность копирования среды для случаев, когда используются источники данных на основе управляемого Microsoft Dataverse хранилища Data Lake или папки Common Data Model.

## <a name="january-2021-updates"></a>Обновления за январь 2021 г.

Обновления в январе 2021 года включают в себя несколько функций, улучшения производительности и исправления ошибок.

#### <a name="extensibility"></a>Расширяемость

- **Расширенная функциональность и повышенная производительность для экспорта SFTP** Теперь вы можете экспортировать все сущности выходных данных из Customer Insights в узел SFTP. Ранее экспорт был ограничен сущностями сегмента. Кроме того, производительность экспорта SFTP позволяет увеличить объем данных за меньшее время, в зависимости от производительности вашего узла SFTP.    
  Для получения дополнительной информации см. [Соединитель для SFTP (предварительная версия)](export-sftp.md).  

#### <a name="segments"></a>Сегменты

- **Предложенные сегменты на основе машинного обучения для улучшения метрик** Появился новый способ обнаруживать и создавать сегменты. Система использует модель ИИ, чтобы предлагать сегменты, которые могут помочь улучшить показатели KPI (меру), которые вы уже отслеживаете. Мы показываем степень влияния выбранных вами атрибутов на меру или другой основной атрибут. Эта информация помогает найти потенциальные сегменты, представляющие возможности.    
  Для получения дополнительной информации см. [Предлагаемые сегменты (предварительная версия)](suggested-segments.md).

#### <a name="data-unification"></a>Объединение данных

- **Улучшенное взаимодействие для соответствия** В области объединения данных обновлено взаимодействие для соответствия. Он позволяет настраивать и просматривать правила сопоставления, включая подробную статистику, чтобы дополнительно объяснить, как работает сопоставление. Существуют варианты отключения правила сопоставления, чтобы оно больше не было активным, при сохранении конфигурации, правила сопоставления перетаскиванием и т. д.
  Дополнительные сведения см. в статье [Соответствие сущностей](match-entities.md).

- **Выходные данные дедупликации из процесса сопоставления доступны как сущность** Выходные данные процесса дедупликации из процесса сопоставления теперь записываются в отдельную сущность для дальнейшего анализа. Эта сущность состоит из полей, используемых в процессе дедупликации, и записи победителя, а также соответствующих альтернативных записей, которые объединяются с записью победителя.
  Для получения дополнительной информации см. [Выходные данные дедупликации как сущность](match-entities.md#deduplication-output-as-an-entity).

#### <a name="system-administration"></a>Системное администрирование

- **Беспрепятственно делитесь данными с Microsoft Dataverse** Теперь вы можете поделиться выходными данными Customer Insights с приложениями Microsoft Dataverse с помощью Microsoft Dataverse Managed Data Lake. Как только вы свяжете среду Dataverse с Customer Insights, вы можете включить совместное использование данных.
  Дополнительные сведения см. в статье [Управление средами](manage-environments.md).
