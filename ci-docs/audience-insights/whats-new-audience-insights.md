# <a name="whats-new-in-the-audience-insights-capability-of-dynamics-365-customer-insights"></a>Что нового в возможности аналитики аудитории в Dynamics 365 Customer Insights

[!INCLUDE [cc-data-platform-banner](../includes/cc-data-platform-banner.md)]

Мы рады объявить о выходе последних обновлений! В этой статье представлены общедоступные предварительные версии функций, улучшения общей доступности и обновления функций. Узнать долговременные планы относительно функций можно из [планов выпуска Dynamics 365 и Power Platform](/dynamics365/release-plans/).

## <a name="june-2021-updates"></a>Обновления от июня 2021 года

Обновления в июне 2021 года включают в себя несколько функций, улучшения производительности и исправления ошибок.

Эта функция позволяет отслеживать подробный ход выполнения, чтобы понять поток процесса и принять меры, если какой-либо шаг требует внимания.

### <a name="extensibility"></a>Расширяемость

- **Экспорт сегментов и других данных в Salesforce Marketing Cloud** Мы расширили наши пункты назначения экспорта, добавив [Salesforce Marketing Cloud](export-salesforce.md). Импорт данных можно полностью автоматизировать в Salesforce и использовать для создания более эффективных маркетинговых кампаний.  
 
- **Экспорт сегментов в ActiveCampaign** Мы расширили наши экспортные пункты назначения, включив в них [ActiveCampaign](export-active-campaign.md). Теперь можно экспортировать сегменты для создания кампаний, проведения маркетинга по электронной почте и работы с определенными группами клиентов в ActiveCampaign.
 
- **Экспорт сегментов в Sendinblue** Мы расширили наши экспортные пункты назначения, включив в них [Sendinblue](export-sendinblue.md). Теперь можно экспортировать сегменты для создания кампаний, проведения маркетинга по электронной почте и работы с определенными группами клиентов с Sendinblue.
 
### <a name="ux-updates"></a>Обновления интерфейса 

- **Новая и улучшенная страница клиентов и страница сведений профиля** Мы переработали страницу «Клиенты» и страницы сведений о профиле, чтобы улучшить интерфейс пользователя и повысить производительность. Эти изменения позволяют просматривать, сортировать, искать и фильтровать клиентов. Фильтры теперь представлены в URL-адресе, чтобы беспрепятственно делиться результатами поиска с другими пользователями. Результаты поиска также можно сохранить как сегмент.    
  На странице сведений для профилей клиентов данные теперь группируются в различных подразделах, таких как демографические данные, идентификаторы и другие атрибуты профиля, для повышения удобочитаемости. Другие разделы на странице сведений о профиле стали более интерактивными. Например, раздел действий теперь позволяет фильтровать и сортировать.

Это новое представление помогает управлять использованием определенного сегмента и адаптировать существующие или создавать новые экспортные данные.

Для получения дополнительной информации см. [Экспорт сегментов в Omnisend](export-omnisend.md)

### <a name="predictions"></a>Прогнозы

- **Отчет об используемости входных данных** Отчет об удобстве использования входных данных предоставляет консолидированное представление об ошибках и предупреждениях, которые могут генерироваться вашими готовыми прогнозами. Также даются рекомендации по повышению производительности модели.    
  Отчет доступен после завершения процесса обучения модели. Он создается для каждой модели отдельно, независимо от того, успешно она завершилась или нет.
  В настоящее время эта функция доступна только для модели оттока транзакций. Дополнительные сведения см. в [Отчет об используемости входных данных](manage-predictions.md#input-data-usability-report).

Отношения теперь организованы в группы: созданные пользователем отношения, системные отношения и унаследованные отношения. Вы также можете экспортировать представление как изображение. ## <a name="april-2021-updates"></a>Обновления за апрель 2021 года

Обновления в апреле 2021 года включают в себя несколько функций, обновления для повышения производительности и исправления ошибок.

### <a name="data-unification"></a>Объединение данных
 
- **Расширенные возможности слияния для объединения данных**    
  
   Теперь у нас есть улучшенный пользовательский интерфейс в конфигурации слияния для процесса объединения данных. Изменения включают интуитивно понятное упорядочивание объединенных полей и подробную статистику по объединенным и отдельным полям.

- **Переупорядочивание сущностей и настройка всех исходных записей в сущности клиента**  
      
   Теперь вы можете изменить порядок и удалить сущности из существующего плана объединения в процессе объединения данных. Это дает гибкость для изменения порядка сущностей в процессе сопоставления в соответствии с потребностями бизнеса. Дополнительно мы разрешаем включать все несовпадающие записи в финальную сущность *Клиент*, которая позволяет им определять свое определение набора данных профиля клиента.

### <a name="enrichments"></a>Обогащения

 - **Новое обогащение: расширенные адреса**    
  
   Мы рады представить новое обогащение для улучшения адресов в ваших данных о клиентах. Адреса в ваших данных могут быть неструктурированными, неполными или неправильными. Эта функция использует модели Microsoft для нормализации и обогащения ваших адресов в формате Common Data Model для большей точности и понимания.
 
   Дополнительные сведения см. в разделе [Пополнение профилей клиентов расширенными адресами](enrichment-enhanced-addresses.md).

- **Опыт управляемой настройки для обогащений**    
  
   Мы пересмотрели интерфейс настройки для обогащений с простым и управляемым интерфейсом. Теперь у вас есть четкий пошаговый процесс создания и редактирования обогащений.
 
   Кроме того, мы разделили конфигурацию подключений для сторонних обогащений, чтобы одно и то же соединение могло использоваться несколькими обогащениями. Только администраторы могут настраивать новые подключения, но созданные подключения доступны как администраторам, так и участникам.    

   Дополнительные сведения см. в разделе [Обзор подключений](connections.md).

- **Несколько обогащений одного типа**    
  
   Теперь мы позволяем пользователям создавать и управлять несколькими обогащениями одного и того же типа обогащения. Например, теперь вы можете создать два отдельных обогащения адресов, чтобы обогатить два разных сегмента клиентов. Ограничения применяются к количеству обогащений одного и того же типа, которые могут быть созданы, и варьируются в зависимости от типа обогащения.
  
   Для получения дополнительной информации см. [Обогащение профилей клиентов](enrichment-hub.md).

## <a name="february-2021-updates"></a>Обновления за февраль 2021 г.

Обновления в феврале 2021 года включают в себя несколько функций, улучшения производительности и исправления ошибок.

#### <a name="extensibility"></a>Расширяемость

- **Экспорт сегментов в AdRoll**

  Мы расширили наши пункты назначения экспорта, включив AdRoll. Теперь вы можете экспортировать сегменты из Customer Insights в аудитории AdRoll и использовать их в качестве базового уровня для своей рекламы. Для получения дополнительной информации см. [Соединитель для AdRoll](export-adroll.md).

#### <a name="segments"></a>Сегменты
 
- **Дублировать сегмент**
  
  Чтобы создать новый сегмент на основе существующего, теперь вы можете дублировать сегмент и редактировать его для дальнейшего уточнения. 

- **Добавить дополнительные атрибуты к сегменту**

  Теперь вы можете включать атрибуты в выходные данные вашего сегмента, даже если эти атрибуты не являются частью профиля клиента. Например, вы можете включить ИД подписок в сегмент, даже если он является частью сущности подписки, которая имеет отношение M:1 с сущностью клиента. Если атрибут принадлежит сущности, связанной с сущностью клиента, вы можете включать эти атрибуты.  

#### <a name="predictions"></a>Прогнозы

- **Создание прогнозных рекомендаций по продуктам**

  Понимание того, что клиенты заинтересованы в покупке, — это одно из главных требований увеличения доходов бизнеса и повышения лояльности клиентов за счет персонализации и взаимодействия. Предоставление рекомендаций по продуктам, которые не соответствуют интересам вашего клиента, может создать ощущение разрыва между клиентом и вашей компанией и, в конечном итоге, ограничить общий потенциальный доход и опыт для клиента. 

  Используя свои собственные данные, теперь вы можете прогнозировать, какие товары ваши клиенты, скорее всего, купят в будущем. Для получения дополнительной информации см. [Прогноз рекомендации продуктов](predict-product-recommendation.md).

#### <a name="system-administration"></a>Системное администрирование

- **Копирование среды поддерживается для разных типов источников данных**

  Администраторы могут копировать конфигурации среды в новую среду в той же организации. Эта функция расширяет функциональность копирования среды для случаев, когда используются источники данных на основе управляемого Microsoft Dataverse хранилища Data Lake или папки Common Data Model.

## <a name="january-2021-updates"></a>Обновления за январь 2021 г.

Обновления в январе 2021 года включают в себя несколько функций, улучшения производительности и исправления ошибок.

#### <a name="extensibility"></a>Расширяемость

- **Расширенная функциональность и повышенная производительность для экспорта SFTP** Теперь вы можете экспортировать все сущности выходных данных из Customer Insights в узел SFTP. Ранее экспорт был ограничен сущностями сегмента. Кроме того, производительность экспорта SFTP позволяет увеличить объем данных за меньшее время, в зависимости от производительности вашего узла SFTP.    
  Для получения дополнительной информации см. [Соединитель для SFTP (предварительная версия)](export-sftp.md).  

#### <a name="segments"></a>Сегменты

- **Предложенные сегменты на основе машинного обучения для улучшения метрик** Появился новый способ обнаруживать и создавать сегменты. Система использует модель ИИ, чтобы предлагать сегменты, которые могут помочь улучшить показатели KPI (меру), которые вы уже отслеживаете. Мы показываем степень влияния выбранных вами атрибутов на меру или другой основной атрибут. Эта информация помогает найти потенциальные сегменты, представляющие возможности.    
  Для получения дополнительной информации см. [Предлагаемые сегменты (предварительная версия)](suggested-segments.md).

#### <a name="data-unification"></a>Объединение данных

- **Улучшенное взаимодействие для соответствия** В области объединения данных обновлено взаимодействие для соответствия. Он позволяет настраивать и просматривать правила сопоставления, включая подробную статистику, чтобы дополнительно объяснить, как работает сопоставление. Существуют варианты отключения правила сопоставления, чтобы оно больше не было активным, при сохранении конфигурации, правила сопоставления перетаскиванием и т. д.
  Дополнительные сведения см. в статье [Соответствие сущностей](match-entities.md).

- **Выходные данные дедупликации из процесса сопоставления доступны как сущность** Выходные данные процесса дедупликации из процесса сопоставления теперь записываются в отдельную сущность для дальнейшего анализа. Эта сущность состоит из полей, используемых в процессе дедупликации, и записи победителя, а также соответствующих альтернативных записей, которые объединяются с записью победителя.
  Для получения дополнительной информации см. [Выходные данные дедупликации как сущность](match-entities.md#deduplication-output-as-an-entity).

#### <a name="system-administration"></a>Системное администрирование

- **Беспрепятственно делитесь данными с Microsoft Dataverse** Теперь вы можете поделиться выходными данными Customer Insights с приложениями Microsoft Dataverse с помощью Microsoft Dataverse Managed Data Lake. Как только вы свяжете среду Dataverse с Customer Insights, вы можете включить совместное использование данных.
  Дополнительные сведения см. в статье [Управление средами](manage-environments.md).
