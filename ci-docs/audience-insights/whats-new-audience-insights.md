## <a name="november-2020-updates"></a>Обновления за ноябрь 2020 г.

Обновления в ноябре 2020 года включают несколько функций, повышение производительности и исправления ошибок.

### <a name="new-and-updated-features-in-november-2020"></a>Новые и обновленные функции в ноябре 2020 г.

#### <a name="data-enrichment"></a>Обогащение данных

- **Перенесите собственные данные обогащения с помощью настраиваемого импорта протокола SFTP**
  
  Настраиваемый импорт протокола SFTP позволяет импортировать данные обогащения, которые не должны проходить процесс объединения данных. Подробнее о пользовательском импорте протокол SFTP.

  Для получения дополнительной информации см. [Обогащение профилей пользователей пользовательскими данными (предварительная версия)](enrichment-SFTP-custom-import.md).
 
- **Обогатите свои данные о клиентах с помощью данных расположения от HERE Technologies**

  С помощью услуг по обогащению данных HERE Technologies вы можете получить более точное представление о местоположении ваших клиентов с помощью нормализации адресов, извлечения широты и долготы и т. д. Узнайте больше об обогащении с помощью HERE Technologies.

  Для получения дополнительной информации см. [Обогащение профилей пользователей с использованием HERE Technologies](enrichment-here.md).

#### <a name="data-unification"></a>Объединение данных

- **Гибкость для включения профилирования данных по выбранным сущностям и полям из вашей учетной записи хранения**

  Вы можете указать, для каких сущностей данных и полей из папки Common Data Model в вашей учетной записи хранения Azure Data Lake вы хотите включить профилирование данных как часть процесса приема данных.

  Для получения дополнительной информации см. [Подключение к папке Common Data Model](connect-common-data-model.md#connect-to-a-common-data-model-folder).

#### <a name="extensibility"></a>Расширяемость

- **Активируйте свои сегменты через Google Ads**

  Экспортируйте сегменты в списки аудитории Google Ads и используйте эти списки для рекламы в поиске Google, Gmail, YouTube и Google Display Network. Подробнее об активации сегментов через Google Ads.

  Для получения дополнительной информации см. [Соединитель для Google Ads](export-google-ads.md).

- **Активируйте свои сегменты через Marketo**

  Экспортируйте сегменты в аудитории Marketo и используйте эти аудитории для автоматизации маркетинга. Подробнее об активации сегментов через Marketo. 

  Для получения дополнительной информации см. [Соединитель для Marketo](export-marketo.md).

- **Активируйте свои сегменты через DotDigital**

  Экспортируйте сегменты в DotDigital и используйте их в маркетинговых целях. Подробнее об активации сегментов через DotDigital. 

  Для получения дополнительной информации см. [Соединитель для DotDigital](export-dotdigital.md).

#### <a name="predictions"></a>Прогнозы

- **Прогноз оттока транзакций**

  Функция прогноза оттока транзакций позволяет вам без помощи специалиста по данным прогнозировать вероятность того, что клиент перестанет покупать продукты или услуги.  Используя оценку прогноза, вы можете комбинировать другую информацию о ваших клиентах, например ценность клиента, для создания сегментов с высоким риском оттока или ценных клиентов. Используйте этот сегмент для прямого воздействия на клиентов посредством маркетинговых мероприятий, поддержки клиентов и других сценариев, чтобы снизить риск оттока.
 
  Настройте определение оттока как временное окно с учетом условий вашего бизнеса, и определите, когда клиенты считаются ушедшими. Например, в продуктовом магазине могут считать клиента ушедшим, если он ничего не покупал в течение последних 30 дней.
 
  По мере того, как вы продолжаете создавать прогноз, мы подскажем, какие данные необходимы, и позволим вам сопоставить данные о своем бизнесе с полями, необходимыми для прогнозирования оттока клиентов. Вы также можете установить расписание для переобучения модели на основе новой информации в вашей системе, чтобы адаптироваться к меняющимся условиям бизнеса.
 
  Для получения дополнительной информации см. [Прогноз оттока транзакций (предварительная версия)](predict-transactional-churn.md).

#### <a name="system-administration"></a>Системное администрирование

- **Выполнить сброс среды**

  Сбросить всё в среде выбранного экземпляра, чтобы начать все заново.

  См. дополнительные сведения о [Сбросить существующую среду](manage-environments.md#reset-an-existing-environment).


- **Подключение к учетной записи хранения Azure Data Lake с помощью субъекта-службы**

  Записывать выходные данные в учетную запись хранения и считывать данные из нее с помощью субъекта-службы Azure. Существующие подключения к учетной записи хранения могут продолжать использовать ключ учетной записи. Они также предлагают вариант обновления для использования субъекта-службы в дальнейшем. Новые подключения будут основаны на методе аутентификации субъекта-службы для вашей учетной записи хранения.

  Для получения дополнительной информации см. [Подключение к учетной записи Azure Data Lake Storage Gen2 с помощью субъекта-службы Azure для аналитики аудитории](connect-service-principal.md).

## <a name="october-2020-updates"></a>Обновления за октябрь 2020 г.

Обновления в октябре 2020 года включают несколько функций, повышение производительности и исправления ошибок.

### <a name="new-and-updated-features-in-october-2020"></a>Новые и обновленные функции в октябре 2020 г.

#### <a name="extensibility"></a>Расширяемость

- **Экспорт в Mailchimp**

Экспортируйте сегменты в существующие списки аудитории в Mailchimp, чтобы обеспечить индивидуальный подход к электронной почте для ваших клиентов.

Для получения дополнительной информации см. [Соединитель для Mailchimp](export-mailchimp.md).

#### <a name="data-enrichment"></a>Обогащение данных

- **Дедупликация исходных записей в сущности Match**

Укажите правила дедупликации для сущностей, используемых в процессе сопоставления, для выявления повторяющихся записей. Объедините их в одну запись и свяжите все исходные записи с этой объединенной записью. Эта дедуплицированная запись будет использоваться в процессе сопоставления между объектами.

Для получения дополнительной информации см. [Определите дедупликацию для соответствующей сущности](match-entities.md#define-deduplication-on-a-match-entity).

#### <a name="system-administration"></a>Системное администрирование

- **Оркестрация: новая функция обновления в объединении**

До сегодняшнего дня, когда вы запускали процесс объединения, система запускала все последующие процессы, которые зависят от слияния и последующих процессов. Теперь вы можете проверить результат процесса объединения (объединенная сущность клиента), прежде чем использовать его в последующей обработке, такой как сегменты или меры.
На странице объединения теперь вы можете выбрать запуск только шага объединения и запускать только этот процесс. Чтобы обновить все последующие процессы, вы можете выбрать запускать объединения и последующие процессы. 

## <a name="september-2020-updates"></a>Обновления за сентябрь 2020 г.

Обновления за сентябрь 2020 года включают в себя несколько функций, обновлений для повышения производительности и исправлений ошибок.

### <a name="new-and-updated-features-in-september-2020"></a>Новые и обновленные функции в сентябре 2020 г.

#### <a name="activities"></a>Операции

- **Интеллектуальный прогноз семантики атрибутов**

Эта новая функция предсказывает семантические типы входных атрибутов, которые передаются в процесс объединения данных. Он использует модели машинного обучения, которые повышают точность и экономят время.

#### <a name="enrichments"></a>Обогащения

- **Обогащение демографических данных из Experian**

Обогащение демографических данных от Experian теперь доступно в предварительной версии. Experian является мировым лидером в области потребительской и деловой кредитной отчетности и маркетинговых услуг. Со [службами Experian по обогащению данных](https://www.experian.com/marketing-services/microsoft?cmpid=ems_web_mci_cdppage) вы можете глубже понять своих клиентов, обогатив профили клиентов демографическими данными, такими как размер семьи, доход и т. д.

Чтобы использовать эту функцию, у вас должна быть активная подписка Experian.

Для получения дополнительной информации см. раздел [Обогащение профилей клиентов демографическими данными из Experian](enrichment-experian.md)


#### <a name="system-administration"></a>Системное администрирование

- **Область сведений о задаче**

Панель сведений о задаче позволяет просматривать сведения о задачах, выполняемых системой. Это удобный способ выявить проблемы с конфигурацией и найти решения.
Просмотрите сообщения об ошибках, чтобы узнать, как вы устраняете потенциальные проблемы.
 
- **Информация об обработке добавлена на дополнительные страницы**

Это улучшение добавляет информацию о статусе ваших сущностей на страницы **Сущности** и **Клиенты**.
 
Кроме того, на обеих страницах вы можете найти подробную информацию о ходе процессов, а также сведения о задачах.

- **Улучшения на странице состояния системы**

Мы улучшили структуру таблицы сведений о статусе в разделе **Система** > **Статус** при просмотре экспорта данных.
 
Кроме того, ошибки в столбце **Сведения** теперь более подробные и полезные. 
 
- **Отмена возвращает задание в предыдущее состояние**

Когда вы отменяете задачу, например, в процессе сопоставления, она возвращается к своему последнему состоянию. Например, если процесс сопоставления завершился вчера, а вы отмените его сегодня, он вернется к вчерашнему успешному состоянию.

#### <a name="enrichment"></a>Обогащение

- **Обогащение данными о компании из Leadspace**
  
  Определите поля в унифицированных профилях клиентов, которые используются для поиска связанных данных компании в Leadspace. После запуска процесса обогащения профили B2B обогащаются дополнительными атрибутами, включая размер компании, местоположение, отрасль и т. д.

  Для получения дополнительной информации см. раздел [Обогащение профилей компаний с помощью Leadspace](enrichment-leadspace.md).

- **Страница центра обогащения**

  Все доступные методы обогащения данных от собственных и сторонних поставщиков обогащения настраиваются в одном месте. Настройка обогащения данных — это простой процесс, которым можно управлять из общего места.    
  Для получения дополнительной информации см. [Обогащение профилей клиентов](enrichment-hub.md).

- **Раздельное обогащение повышения узнаваемости бренда и интересов**

  Повышение узнаваемости брендов и интересов теперь доступно как два независимых обогащения. Раздельное обогащение дает вам возможность настраивать и управлять ими индивидуально, в зависимости от требований или потребностей вашего бизнеса.    
  Дополнительные сведения см. в разделе [Обогащение профилей клиентов с повышением узнаваемости бренда и интересов](enrichment-microsoft-graph.md).

#### <a name="extensibility"></a>Расширяемость

- **Интерактивные URL-адреса для унифицированных действий в надстройке карточек клиента Dynamics 365**

  Унифицированные действия в надстройке карточки клиента теперь отображают интерактивные URL-адреса, если такие URL-адреса были определены во время настройки действий.    
  Дополнительные сведения см. в статье [Надстройка карточки клиента](customer-card-add-in.md).

- **Сродство с брендом и интересами, доступное в надстройке карточек клиента Dynamics 365**

  Дополнительные сведения см. в статье [Надстройка карточки клиента](customer-card-add-in.md).

#### <a name="predictions"></a>Прогнозы

- **Прогноз оттока подписок**

  Следуйте инструкциям, чтобы создать прогноз оттока в областях подписки, таких как облачные сервисы, членство клиентов и в других секторах. 

  Функция прогноза оттока подписок позволяет прогнозировать вероятность того, что клиент прекратит использование продуктов или услуг, основанных на подписке, без привлечения специалиста по данным. Используя оценку прогноза, вы можете комбинировать другую информацию о своих клиентах для создания сегментов с высоким риском оттока. С помощью этого сегмента вы можете напрямую ориентироваться на клиентов в маркетинге, поддержке клиентов и в других сценариях, чтобы снизить риск оттока конкретных клиентов, повысить доход и снизить затраты.

  В рамках опыта вы можете настроить определение оттока как временного окна, специфичного для вашего бизнеса. Например, компания, занимающаяся потоковым видео с ежемесячной подпиской, может захотеть рассмотреть клиента, который ушел через 15 дней после истечения срока действия подписки.

  По мере того, как вы продолжите рассматривать прогноз, мы покажем, какие данные необходимы, и позволим вам сопоставить данные о вашем бизнесе с полями, необходимыми для прогнозирования оттока клиентов. По мере изменения бизнес-информации вы также можете установить расписание для повторного обучения по новой информации в вашей системе, чтобы адаптироваться к меняющимся условиям бизнеса.    
  Для получения дополнительной информации см. раздел [Прогноз оттока подписок (предварительная версия)](predict-subscription-churn.md).

#### <a name="segments"></a>Сегменты

- **Найти похожих клиентов**
  
  Найдите похожих клиентов в своей клиентской базе с помощью искусственного интеллекта. Модель машинного обучения с двоичной классификацией присваивает оценку сходства клиентам в расширенном сегменте. Оценка основана на сходстве с клиентами в исходном сегменте. В зависимости от оценки сходства профили клиентов добавляются во вновь созданный сегмент.

  Иногда в цифровом маркетинге это называется моделированием сходства, при этом используется модель искусственного интеллекта, чтобы помочь найти клиентов, похожих на другой сегмент ваших клиентов, с учетом дополнительных атрибутов. Это не только позволяет выбирать атрибуты, но также позволяет указать максимальное количество клиентов, которые должны быть в этом новом сегменте. Затем модель ИИ вычислит оценки сходства для каждого клиента на основе выбранных вами атрибутов и найдет клиентов с более высоким средним показателем сходства. Результирующий сегмент будет включать клиентов, которые похожи на клиента из вашего исходного сегмента.    
  Дополнительные сведения см. в разделе [Похожие клиенты](find-similar-customer-segments.md).

- **Перекрытие сегментов и дифференциаторы**

  Перекрытие сегментов позволяет увидеть, сколько и какие клиенты являются общими для двух или более сегментов. Например, как сегмент клиентов с высокими тратами перекрывается с сегментом клиентов с высоким уровнем удовлетворенности или как сегмент оттока клиентов перекрывается с сегментом клиентов с низким уровнем удовлетворенности. Кроме того, вы можете проанализировать, как изменяется перекрытие, на основе дополнительного атрибута по вашему выбору.

  Дифференциаторы сегментов показывают, что отличает один сегмент от остальных ваших клиентов или от другого сегмента. Все, что вам нужно сделать, это определить сегмент, и система определит атрибуты профиля и показатели, которые отличают сегмент, в виде ранжированного списка отличительных признаков — от самого сильного дифференцирующего признака до самого слабого.    
  Дополнительные сведения см. в разделе [Аналитические сведения о сегментах (предварительная версия)](segment-insights.md).

  Определите расписание для активации или деактивации сегмента.    
  Дополнительные сведения см. в статье [Управление существующими сегментами](segments.md#manage-existing-segments).
